# Security Policy

## Supported Versions

We actively provide security patches and updates for the following product versions:

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |
| < v1.0  | :x:                |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

If you discover a security vulnerability, we ask that you report it privately so that we can address and resolve the concern before exposing details to malicious entities.


### What to Include in a Report
- A detailed description of the potential vulnerability.
- Proof of Concept (PoC) scripts, steps, or requests needed to reproduce the behavior.
- Potential impact assessments (e.g., local database access, cross-site scripting risks).

We aim to acknowledge receipt of security reports within 48 hours, and provide a resolution outline or vendor fix within 14 business days. Thank you for helping keep CommitCraft secure!
